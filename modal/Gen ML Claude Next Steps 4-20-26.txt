  improv_loop.py build order

  The client and server are done. The remaining pieces in the order to build them:

  Step 1 — Audio device detection (src/audio_devices.py)
  List CABLE Output index and default output device index. Run once, save to config.

  Step 2 — Loop capture (src/loop_capture.py)
  sounddevice.InputStream reading from CABLE Output at 48kHz stereo. Accumulates into a ring buffer. On command, snaps exactly beats × (60/bpm) × 48000 samples.

  Step 3 — Audio playback + mixer (src/audio_mixer.py)
  sounddevice.OutputStream callback that reads from loop buffer + voice output queues, mixes (numpy add), peak-normalizes, writes to speakers. This is what the user
  hears.

  Step 4 — PBF4 MIDI (src/midi_controller.py)
  mido.open_input() callback that maps CC numbers to GenerationParams fields. Requires the CC numbers you discover in hardware setup above.

  Step 5 — Timing engine (src/timing_engine.py)
  Absolute-time loop clock (time.perf_counter). Fires a callback at every pass boundary. Drives the state machine.

  Step 6 — Main orchestrator (improv_loop.py)
  Wires everything together: state machine (IDLE → COUNTDOWN → RECORDING → PLAYING), MagentaRTClient.generate_pass() called once per pass boundary in an asyncio task,
  results queued for the mixer.
